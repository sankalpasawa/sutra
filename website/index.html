One command. Your AI assistant works like a senior engineer.

Sutra is a plugin for Claude Code that makes your AI think before it types — classify what you asked for, estimate the work, surface the plan, then act. No more 50,000-token sessions to fix a typo.

Install — one command (works on macOS & Linux)
curl -fsSL https://sutra.os/install.sh | bash
Installs Claude Code if you don't have it, adds the Sutra marketplace, installs the plugin, and writes the permission allowlist. Idempotent — safe to re-run. After it finishes, open Claude Code in any project and run /core:start. Prefer manual install? →
What you get

Five things change in every Claude Code session.

01 — Routing

Every message gets classified

Claude doesn't just react — it labels what you said (a task, a question, a correction, a new direction) before doing anything. Stops misrouted work at the front door.

02 — Depth

Every task gets a depth rating

Before any code edit, Claude rates the task 1-5 and shows you the time / file count / cost estimate. You see the bill before you pay.

03 — Plan first

Every action gets a plan you can stop

Before any Edit / Write / Bash call, Claude shows a Blueprint — what it's about to do, the steps, when it would stop. You catch wrong turns in seconds.

04 — Readable

Output formatted for skimming

Tables over walls of prose. Numbers over adjectives. Decisions in boxes. Designed to be scanned in 10 seconds, not read in 2 minutes.

05 — Permissions

Sutra's own commands auto-approved

The plugin handles its own permission prompts — first session goes from ~13 prompts down to ≤2. You spend time on your code, not clicking "allow".

06 — Trace

One-line trace per response

Every reply ends with a tiny breadcrumb showing what Claude did under the hood — input → routing → plan → action → result. No guessing what just happened.

Privacy

Your code stays on your machine.

Sutra captures nothing about what you're building, by default. No file contents. No prompts. No commit messages. No code. Only tiny anonymous signals — "a hook ran," "a session ended" — kept locally in ~/.sutra/. Nothing is transmitted unless you explicitly opt in by setting SUTRA_LEGACY_TELEMETRY=1. A plain-English privacy sheet (~/.sutra/PRIVACY.md) lands on your disk the first time you run it — one page, no legalese, read it in two minutes. Delete the folder to wipe it all.

Manual install (if you prefer)

Three commands. One minute.

The one-command installer runs all of these for you. If you'd rather see each step:

Step 1 — Install Claude Code (skip if you already have it)

curl -fsSL https://claude.ai/install.sh | bash
Or download the desktop app, or use the web app. Verify with claude --version.

Step 2 — Add the Sutra marketplace + install the plugin

claude plugin marketplace add sankalpasawa/sutra
claude plugin install core@sutra
Two commands. Restart Claude Code (or run /reload-plugins) to pick it up.

Step 3 — Activate in a project

cd your/project && claude
Then inside Claude Code: /core:start. Idempotent — safe to re-run.
Platform compatibility. Skills work everywhere Claude Code works (terminal, Mac/Windows desktop, web, VS Code / JetBrains). Enforcement hooks need bash — full strength on macOS / Linux terminal + Mac desktop + IDE extensions. Web app: skills work, hooks sandboxed. Windows: install on WSL2.
