Get started

Never used Claude Code? Here's the full path.

Sutra is a plugin for Claude Code, Anthropic's AI coding environment. If you've never used Claude Code, follow these four steps. If you already have Claude Code running, jump to step 3.

1

Install Claude Code

Claude Code runs in the terminal, Mac/Windows desktop app, web (claude.ai/code), and as VS Code / JetBrains extensions. Pick your surface and install.

Terminal (Mac / Linux):

curl -fsSL https://claude.ai/install.sh | bash

After install, add Claude Code to your PATH:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Verify with claude --version — expect version ≥ 2.1.116.

Or download the desktop app, or use the web app — same Sutra install flow.

2

Sign in + verify

Run claude — it opens the sign-in flow. Works with an Anthropic account or Claude.ai login.

1 min
Coming in v1.8: a single-command install — curl -fsSL https://sutra.os/install.sh | bash. For now, the 4-step path below.
3

Add the Sutra marketplace + install the plugin

Two commands in your terminal (or inside any Claude Code session with /plugin):

claude plugin marketplace add sankalpasawa/sutra
claude plugin install core@sutra

Restart Claude Code, or run /reload-plugins inside your session to pick up the plugin. The five core skills + onboarding commands are now active in every session.

4

Activate + do your first governed task

In any project directory, open Claude Code and type:

/core:start

One command. Writes your install identity, enables telemetry, prints the activation banner, writes a depth marker. Then ask Claude to do anything — you'll see the input-routing block and depth-estimation block before any edit. Five skills + governance enforcement now active.

What the OS provides

Governance, not just tooling.

Most AI dev tools help you type faster. Sutra changes how work is classified, scoped, and enforced across every session — for every company in your portfolio.

01

Input Routing

Every user message is classified — direction, task, feedback, new concept, or question — before any action. Stops misrouted work at the front door.

02

Depth & Estimation

Every task gets a depth rating (1-5) and a cost/effort/impact estimate before it starts. Post-task actuals feed an auto-calibrating engine.

03

Coverage Engine

Runtime process coverage — 26 trackable methods across 6 categories — measures whether the expected Sutra steps actually fired per task, per depth.

04

Policy-Coverage Gate

PROTO-017: every policy marked HARD must have a compiled blocking hook and a regression test. If it can't be enforced, it can't be policy.

05

Auto-Propagation

PROTO-018: when Sutra bumps version, every client company in the registry is re-manifested, re-pinned, and re-installed. No drift between the OS and the companies that run on it.

06

Initiative Charters

Cross-cutting outcome goals (Tokens, Speed) with DRI, contributors, quarterly OKRs. Charters sit alongside unit definitions, compound across the portfolio.

How Sutra thinks

Company-first, not code-first

Most AI tools help you write code faster. Sutra helps you build a company. The difference: code is a means, not the end. Sutra makes sure you're building the right thing before you build it fast.

Sense

Understand the situation

Check production health. Review learnings from past sessions. Understand what's working and what's not.

Shape

Turn fuzzy into clear

Brainstorm, research, carve scope. Every idea gets stress-tested against market reality.

Decide

Commit or kill

Architecture review. Design review. Second opinions. No building without commitment.

Build

Execute with quality

Plan. Execute in waves. Test. Fix. Review code. Ship. Monitor. Learn.
