What the OS provides

Governance, not just tooling.

Most AI dev tools help you type faster. Sutra changes how work is classified, scoped, and enforced across every session — for every company in your portfolio.

01

Input Routing

Every user message is classified — direction, task, feedback, new concept, or question — before any action. Stops misrouted work at the front door.

02

Depth & Estimation

Every task gets a depth rating (1-5) and a cost/effort/impact estimate before it starts. Post-task actuals feed an auto-calibrating engine.

03

Coverage Engine

Runtime process coverage — 26 trackable methods across 6 categories — measures whether the expected Sutra steps actually fired per task, per depth.

04

Policy-Coverage Gate

PROTO-017: every policy marked HARD must have a compiled blocking hook and a regression test. If it can't be enforced, it can't be policy.

05

Auto-Propagation

PROTO-018: when Sutra bumps version, every client company in the registry is re-manifested, re-pinned, and re-installed. No drift between the OS and the companies that run on it.

06

Initiative Charters

Cross-cutting outcome goals (Tokens, Speed) with DRI, contributors, quarterly OKRs. Charters sit alongside unit definitions, compound across the portfolio.

How Sutra thinks

Company-first, not code-first

Most AI tools help you write code faster. Sutra helps you build a company. The difference: code is a means, not the end. Sutra makes sure you're building the right thing before you build it fast.

Sense

Understand the situation

Check production health. Review learnings from past sessions. Understand what's working and what's not.

Shape

Turn fuzzy into clear

Brainstorm, research, carve scope. Every idea gets stress-tested against market reality.

Decide

Commit or kill

Architecture review. Design review. Second opinions. No building without commitment.

Build

Execute with quality

Plan. Execute in waves. Test. Fix. Review code. Ship. Monitor. Learn.
