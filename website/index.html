What a Chief of Staff actually does

Three jobs. Every session.

Not a faster AI. A disciplined one. The kind that turns a 50,000-token tangent into a one-line fix.

01

Reads what you actually meant.

Direction, question, correction, or task — each gets routed differently. "Fix this" no longer becomes a refactor.

02

Shows the bill first.

Time, files touched, dollar cost — all visible before any edit. You stop a 30-min job before it starts when you wanted a 30-second fix.

03

Surfaces every decision.

One-screen plan before any action — not after 50 tool calls. You redirect in seconds, not after the damage.

Privacy

Your code stays on your laptop.

No-content guarantee
Sutra captures nothing about what you're building, by default. No file contents. No prompts. No code. Only anonymous signals — kept locally in ~/.sutra/. Nothing transmitted unless you opt in. Delete the folder to wipe it all.

Native. The runtime that runs a whole company.

One founder. One LLM. Everything else autonomous. Native is the V2.4 Sutra Engine — 4 primitives, 6 laws, decision provenance built in. Ships on commodity infrastructure (Anthropic Claude SDK, Temporal, OPA, OpenTelemetry) with ~1,700 lines of Sutra glue.

curl -fsSL sankalpasawa.github.io/sutra/native/install.sh | bash
Creates ./sutra/native/ in your current directory · ~5 MB engine source · sparse-clones github.com/sankalpasawa/sutra · runs npm install · registers Sutra marketplace with Claude Code if present · re-run safe (existing installs at the same version reuse silently; upgrades and overwrites prompt before mutating).
