# Asawa Inc. — Research Synthesis

## Sources Reviewed
45+ sources across three research tracks:
- **Track 1**: 10 organizational frameworks (Laloux, McChrystal, Dalio, Hastings, Ries, Catmull, Collins, Ismail, Spotify, Beer)
- **Track 2**: 15 books on org design (+ Grove, Doerr, Singer, Slootman, Bryar/Carr, Robertson, Kegan/Lahey, Horowitz)
- **Track 3**: 15 AI-native/holding company structures (Anthropic, OpenAI, Stripe, Linear, Notion, Berkshire, Alphabet, Amazon, Apple, DAOs, agent swarms research)

## The 15 Operating Principles (distilled from all sources)

| # | Principle | Sources | Practice |
|---|-----------|---------|----------|
| 1 | Shared Consciousness as Infrastructure | McChrystal, Dalio, Doerr | Real-time shared knowledge layer all agents/humans access |
| 2 | Advice Process for All Decisions | Laloux, Robertson, Hastings | No approvals needed. Seek advice, then decide. Log everything. |
| 3 | Appetite-Shaped Time-Boxed Ventures | Singer, Ries, Slootman | 6-week cycles. "How much is this worth?" not "How long will it take?" |
| 4 | Flywheel-Driven Focus | Collins, Slootman | Max 3 active ventures. All must accelerate the flywheel. |
| 5 | Build-Measure-Learn at Machine Speed | Ries, Grove, Slootman | Weekly pivot-or-persevere. Innovation accounting. |
| 6 | Task-Relevant Maturity for Agent Autonomy | Grove, Hastings, Horowitz | TRM score per agent per task. Low=guardrails, High=autonomous. |

| 8 | Radical Candor via Braintrust | Catmull, Hastings, Dalio | Bi-weekly peer review. Candor mandatory. No authority. |
| 9 | Single-Threaded Human Leadership | Bryar/Carr, Horowitz, Collins | One dedicated human per venture. No shared leaders. |

## The Core Insight for Asawa Inc.

Asawa Inc. is a **programmable holding company**. Unlike Berkshire Hathaway (where culture spreads through annual letters and trust), Asawa Inc. can literally encode its operating principles into its workforce (AI agents). This collapses the culture-to-execution gap from months to milliseconds.

Three things make this novel:
1. **Culture is code** — principles embedded in agent system prompts, not posters on walls
2. **Governance is automated** — rule-based triggers (from Anthropic's RSP model), not committee meetings
3. **Infrastructure is the moat** — shared agent platform makes the portfolio more than sum of parts

## Recommended Holding Company Structure

```
ASAWA INC. (Holding Company — 3-5 humans)
│
├── IDENTITY (System 5 — Beer's VSM)
│   Purpose, values, principles document (machine-readable)
│   Founder sets direction. Principles guide everything.
│
├── INTELLIGENCE (System 4)
│   Market scanning agents, competitive analysis, trend detection
│   Surfaces opportunities for new portfolio companies
│
├── SHARED INFRASTRUCTURE (System 2)
│   ├── Agent Platform (model routing, orchestration, memory, tools)
│   ├── Financial/Legal (accounting, compliance, entity management)
│   ├── Knowledge Base (cross-portfolio learning, institutional memory)
│   └── Governance Engine (principles-as-code, automated guardrails)
│
├── OVERSIGHT (System 3)
│   OKR dashboards, kill criteria tracking, financial health
│   Automated. Humans review weekly.
│
└── PORTFOLIO COMPANIES (System 1 — each independently viable)
    ├── DayFlow (personal OS — mobile app)
    ├── [Future Company B]
    └── [Future Company C]

    Each company has:
    ├── One single-threaded human leader
    ├── Configured agent swarm (inherits shared infrastructure)
    ├── Own practices (product, design, eng, etc.)
    ├── Own OKRs, own flywheel, own P&L
    └── Reports to holding via defined API contract
```

## How Decisions Flow

| Decision Type | Who Decides | Process |
|--------------|------------|---------|
| Day-to-day execution | Agents autonomously | TRM-scored autonomy |
| Feature prioritization | Portfolio company lead | Advice process + RICE |
| Spending < $500 | Anyone (advice process) | Log decision, seek advice |
| Spending > $500 | Portfolio company lead | Advice process + log |
| New venture / kill venture | Founder (CEO) | PR/FAQ + Braintrust + data |
| Capital allocation | Founder | Quarterly review with dashboards |
| Principles changes | Founder | Pain + reflection cycle |
| Agent system prompt changes | Portfolio lead + founder | Version controlled, tested |

## What Each Portfolio Company Inherits from Holding

1. The 15 operating principles (as code in their agent prompts)
2. Shared agent infrastructure (models, tools, memory systems)
3. Financial/legal back office
4. Governance engine (automated compliance, kill criteria tracking)
5. Cross-portfolio knowledge base

## What Each Portfolio Company Owns Independently

1. Product decisions (what to build, for whom)
2. Go-to-market (pricing, channels, positioning)
3. Team composition (which agents, how configured)
4. Customer relationships
5. P&L accountability
